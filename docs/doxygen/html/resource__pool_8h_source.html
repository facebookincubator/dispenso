89 
90  // There are three reasons we create our own buffer and use placement new:
91  // 1. We want to be able to handle non-movable non-copyable objects
92  // * Note that we could do this with std::deque
93  // 2. We want to minimize memory allocations, since that can be a common point of contention in
94  // multithreaded programs.
95  // 3. We can easily ensure that the objects are cache aligned to help avoid false sharing.
